Documentación del Proyecto DSL

Descripción

Este proyecto implementa un lenguaje específico de dominio (DSL) que permite traducir instrucciones en USQL (una variante personalizada de SQL) a SQL. Incluye una consola interactiva y una API fluida para facilitar el uso del lenguaje.

Instrucciones de Uso

Ejecutar los Tests Unitarios

Para ejecutar todos los tests unitarios, utilice el siguiente comando:

python3 -m unittest test_usql.py

Para ejecutar un test específico, use:

python3 -m unittest test_usql.TestUsql.test_<nombre_del_test>

Ejecutar la Consola Interactiva

Para iniciar la consola interactiva que traduce código USQL a SQL, ejecute:

python3 main.py

Ingrese sus instrucciones en USQL y, para salir, escriba salir.

Estructura del Proyecto

  • main.py: Programa principal que inicia la consola interactiva.
  • lexer.py: Archivo que contiene el analizador léxico de USQL.
  • parser.py: Archivo con el analizador sintáctico que convierte USQL en estructuras internas.
  • translator.py: Convierte estructuras de USQL en comandos SQL.
  • fluent_api.py: Implementa la FluentAPI para construir comandos USQL de manera fluida.
  • test_usql.py: Contiene los tests unitarios para validar la funcionalidad del DSL.
  • keyword_mapping.py: Mapea las palabras clave de USQL a SQL.

Documentación de Funciones Principales

  • lexer.py: Define las reglas para descomponer el código USQL en tokens individuales.
  • parser.py: Contiene funciones para analizar la estructura gramatical de las instrucciones USQL.
  • translator.py: Funciones que traducen la estructura interna de USQL a sintaxis SQL válida.
  • fluent_api.py: Proporciona una API fluida para facilitar la creación de sentencias USQL de manera programática.
  • test_usql.py: Incluye tests que cubren los casos de uso principales y prueba la integración de la FluentAPI y la consola interactiva.
